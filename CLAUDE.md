# CLAUDE.md

## 强制规范

- 使用中文输出所有回答
- 提交代码时，遵循[GIT.md](docs/claude/GIT.md) 中的规范
- 更新模块说明（docs/module/ 目录下的文件）时，遵循[MODULE.md](docs/claude/MODULE.md) 中的规范

## 项目概述

将网页 HTML 转换为可读 Markdown 格式的命令行工具

**核心架构**：`cli.js`（命令行入口）→ `index.js`（转换核心）→ `@mozilla/readability` + `turndown`（依赖库）

## 工作要求

### 行为约束

**禁止主动执行以下操作，除非得到用户明确指令**

- **Git 提交** - 不主动提交代码
- **测试运行** - 不主动运行测试
- **服务启动** - 不主动启动本地测试服务
- **推送远程** - 提交后不推送到远程仓库
- **依赖安装** - 不主动安装依赖
- **文件创建** - 不创建不必要的文件（临时文档、临时测试代码文件、配置文件、日志等）

**注意事项**

- 用户的明确指令只对下一次操作生效，操作完成后回到未收到指令状态

### 设计规范

- **最小化改动** - 修改功能时只改相关文件，涉及其他文件需先询问用户
- **KISS原则** - 保持简单、直接，避免过度设计
- **DRY原则** - 避免重复代码，保持代码简洁；重复错误出现3次时，求助用户或回到更早的阶段重新思考解法

### 流程规范

- **文档同步** - 修改代码后同步更新相关文档
- **询问确认** - 有多种实现方式时，先询问用户偏好

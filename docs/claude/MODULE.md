# 模块文档格式标准

本文档定义 `docs/module/` 目录下功能模块说明文档的格式规范。

## 格式定义

1. 开头
    - 格式
    ```markdown
    # 模块名称10字以内
   
    ```
2. 概述
    - 格式
    ```markdown
    ## 概述
    
    模块功能描述，1-2 句话说明模块做什么，解决什么问题
   
    ```
3. 核心组件
    - 格式
    ```markdown
    ## 核心组件

     | 类/函数     | 文件  | 说明              |
     |----------|-----|-----------------|
     | `类名/函数名` | 文件名 | 组件功能简短说明(20字以内) |
   
    ```

4. 处理流程
    - 格式

    ```markdown
    ## 处理流程
    
    ### 流程1
   
    说明流程1的总体处理步骤
   
    ### 流程2
   
    说明流程2的总体处理步骤
   
    ```

## 特别说明

- 严格按照上述规范格式，不要添加其他章节

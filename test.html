JavaScript 异步编程指南

发布于 2024年1月15日 | 作者: 张三

JavaScript 是一门单线程语言,但它可以通过异步编程来处理耗时操作,避免阻塞主线程。本文将深入讲解 JavaScript 中的异步编程模式。更多资料请参考 MDN Web Docs 和 JavaScript.info。

回调函数

回调函数是最早的异步编程解决方案。一个函数可以作为参数传递给另一个函数,并在适当的时候被调用。参见 Node.js 官方文档 了解更多。

function fetchData(callback) {
    setTimeout(() => {
        callback('数据加载完成');
    }, 1000);
}

fetchData((result) => {
    console.log(result);
});

Promise

Promise 是 ES6 引入的异步编程解决方案,它表示一个异步操作的最终完成或失败。Promise 有三种状态:pending、fulfilled 和 rejected。推荐阅读 Promise A+ 规范。

const promise = new Promise((resolve, reject) => {
    setTimeout(() => {
        resolve('成功');
    }, 1000);
});

promise.then(result => {
    console.log(result);
}).catch(error => {
    console.error(error);
});

Async/Await

async/await 是 ES2017 引入的语法糖,它基于 Promise,让异步代码看起来像同步代码,更加清晰易读。

async function getData() {
    try {
        const result = await fetchData();
        console.log(result);
    } catch (error) {
        console.error(error);
    }
}

总结

JavaScript 异步编程经历了从回调函数到 Promise,再到 async/await 的演进。选择合适的方案可以让代码更加清晰和易于维护。
